# README2

## 작업 진행 순서 정리

이 문서는 현재 프로젝트에서 지금까지 진행한 작업을 시간순으로 정리한 기록입니다.

### 1. 협업용 저장소 초기화
- GitHub 원격 저장소와 연결된 협업용 레포지토리를 준비했습니다.
- 초기 커밋과 원격 푸시를 진행해 팀원들이 같은 기준점에서 시작할 수 있게 만들었습니다.
- 이후 개인 작업 브랜치를 만들어 기능 단위로 분업할 수 있는 흐름을 잡았습니다.

### 2. 프로젝트 방향 전환
- 기존 실시간 협업 에디터 방향에서 벗어나, 미니 프로젝트 요구사항에 맞는 포켓몬 기반 웹 프로젝트로 전환했습니다.
- 사용 기술 스택은 다음 기준으로 정리했습니다.
  - 프론트엔드: HTML, CSS, Bootstrap, JavaScript, jQuery, AJAX
  - 백엔드: Python, Flask, Jinja2
  - 데이터 저장: MongoDB 대응 구조 또는 로컬 저장 대체 구조

### 3. Flask 프로젝트 기본 구조 생성
- `app.py`를 기준으로 Flask 앱을 실행할 수 있는 기본 구조를 만들었습니다.
- `templates/`, `static/`, `services/`, `data/` 디렉터리를 정리했습니다.
- 실행을 위한 `requirements.txt`도 추가했습니다.

### 4. 블루프린트 구조 적용
- 라우트를 한 파일에 몰지 않기 위해 블루프린트를 도입했습니다.
- `blueprints/` 또는 기능 모듈 쪽으로 라우트를 분리해 관리할 수 있는 형태로 바꿨습니다.
- 이후 기능이 늘어나더라도 각 기능을 독립적으로 관리할 수 있도록 구조를 잡았습니다.

### 5. 기능별 협업 구조로 리팩터링
- 4명이 충돌을 줄이며 작업할 수 있도록 기능 단위 구조로 다시 정리했습니다.
- 핵심 아이디어는 각자가 프론트와 백엔드를 하나의 기능 단위로 같이 작업하고, 마지막에 합치는 방식입니다.
- 이를 위해 다음 구조를 도입했습니다.
  - `features/<기능명>/backend.py`
  - `templates/features/<기능명>/template_fragment.html`
  - `static/features/<기능명>/frontend.js`
  - `static/features/<기능명>/frontend.css`
- `feature_registry.py`를 통해 기능을 등록하고 불러오는 방식으로 바꿨습니다.

### 6. 포켓몬 도감 기능 정비
- 메인 화면에서 포켓몬 도감을 볼 수 있도록 도감 페이지를 구성했습니다.
- 포켓몬 카드, 상세 모달, 타입 표시, 지역 구분, 검색/필터 기능을 정리했습니다.
- 포켓몬 클릭 시 상세 정보를 모달에서 볼 수 있도록 만들었습니다.

### 7. 인코딩 문제 해결
- JSON, Python, HTML 파일에서 발생하던 한글 인코딩 문제를 여러 차례 수정했습니다.
- UTF-8 / UTF-8 BOM 문제로 발생하던 오류를 정리했고, 파일 읽기 인코딩도 맞췄습니다.
- 화면에 `????`처럼 보이던 깨진 문구들도 JS와 템플릿에서 정상 한글로 수정했습니다.

### 8. 도감 성능 개선
- 처음에는 모든 포켓몬이 한 번에 렌더링되어 페이지 반응이 느렸습니다.
- 이를 개선하기 위해 지역별 아코디언 구조를 도입했습니다.
- 각 지방 박스는 기본적으로 접힌 상태로 시작하고, 펼칠 때만 카드 DOM을 생성하도록 바꿨습니다.
- 초기 로드 시에는 기본 포켓몬만 먼저 불러오고, 폼/메가/거다이맥스는 필요할 때 추가 로드하도록 변경했습니다.

### 9. 포켓몬 키워드 배경화면 생성 기능 추가
- 새 기능으로 `pokemon-keyword-wallpaper`를 만들었습니다.
- 포켓몬 상세 모달에서 배경화면 생성기로 이어질 수 있도록 연결했습니다.
- 키워드를 입력하면 분위기에 맞는 배경을 코드 기반으로 조합하고, PNG로 저장할 수 있게 했습니다.
- 이후 상세 모달이 너무 커지는 문제를 피하기 위해 배경화면 생성은 별도 흐름으로 분리했습니다.

### 10. 로그인 기능 반영
- Google OAuth 기반 로그인 흐름을 프로젝트에 연결했습니다.
- 로그인 / 로그아웃 / 로그인 상태 표시가 가능하도록 만들었습니다.
- 도감 페이지와 팀 빌더 페이지에서 로그인 상태에 따라 버튼과 안내 문구가 달라지도록 수정했습니다.

### 11. 팀 빌더 기능 추가
- 포켓몬 데이터를 바탕으로 자동 추천 팀을 만들어 주는 팀 빌더 페이지를 구성했습니다.
- 추천 스타일 예시는 다음과 같습니다.
  - 공격형 팀
  - 귀여운 팀
  - 물 타입 중심 팀
  - 전설 제외 팀
  - 초보자 추천 팀
- 추천 결과는 6마리 팀 카드와 시너지 설명으로 렌더링되도록 구현했습니다.

### 12. 내 저장 팀 기능 추가
- 로그인한 사용자가 추천 팀을 저장할 수 있도록 팀 저장 기능을 추가했습니다.
- 백엔드에는 다음 API를 만들었습니다.
  - `GET /api/pokedex/team-builder/teams`
  - `POST /api/pokedex/team-builder/teams`
  - `DELETE /api/pokedex/team-builder/teams/<team_id>`
- 저장 데이터는 현재 로컬 JSON 저장 구조를 통해 관리합니다.
- 팀 빌더 페이지에서 팀 이름 입력 후 `내 팀으로 추가` 버튼으로 저장할 수 있습니다.
- 저장된 팀은 `내 저장 팀` 목록에서 다시 확인하고 삭제할 수 있습니다.

### 13. 팀 저장/삭제 흐름 수정
- 처음에는 추천 팀이 화면에만 렌더링되고 저장용 상태에는 반영되지 않아 저장이 되지 않는 문제가 있었습니다.
- 이를 수정해서 추천 결과가 실제 저장 상태와 연결되도록 바꿨습니다.
- 저장 목록에서 깨져 보이던 한글 문구도 함께 수정했습니다.
- 저장된 팀 카드 우측 상단에서 삭제가 가능하도록 정리했습니다.

### 14. 도감에서 팀 후보 담기 기능 추가
- 도감 상세 모달에서 포켓몬을 바로 `팀에 추가`할 수 있는 버튼을 만들었습니다.
- 이 기능은 브라우저 로컬 저장소를 이용해 팀 후보를 최대 6마리까지 담도록 구현했습니다.
- 중복 포켓몬은 추가되지 않게 막았고, 가득 찼을 때도 상태 문구를 표시합니다.

### 15. 도감 후보와 팀 빌더 연결
- 도감에서 담은 포켓몬은 바로 `내 저장 팀`으로 들어가는 것이 아니라, 임시 팀 후보로 보관되도록 설계했습니다.
- 이를 보완하기 위해 팀 빌더 페이지에 `도감에서 담은 팀 후보` 영역을 추가했습니다.
- 여기서 할 수 있는 동작은 다음과 같습니다.
  - 도감에서 담은 후보 확인
  - 팀 빌더로 불러오기
  - 후보 비우기
- 불러온 뒤 팀 이름을 정하고 `내 팀으로 추가`를 눌러야 최종적으로 `내 저장 팀` 목록에 들어갑니다.

## 현재 사용 흐름

### 도감
1. 메인 페이지에서 포켓몬 도감을 연다.
2. 지역별 아코디언을 펼쳐 포켓몬을 본다.
3. 포켓몬 카드를 클릭해 상세 정보를 확인한다.
4. 상세 모달에서 배경화면 생성 또는 팀 후보 추가를 할 수 있다.

### 팀 빌더
1. 팀 빌더 페이지로 이동한다.
2. 자동 추천 팀을 만들거나, 도감에서 담아둔 팀 후보를 불러온다.
3. 팀 이름을 입력한다.
4. `내 팀으로 추가`를 눌러 저장한다.
5. `내 저장 팀`에서 다시 확인하거나 삭제한다.

## 참고
- 도감에서 `팀에 추가`는 임시 후보 저장입니다.
- `내 저장 팀`은 로그인 계정 기준의 최종 저장 목록입니다.
- 따라서 도감에서 담은 뒤 팀 빌더에서 불러와 저장하는 단계가 한 번 더 필요합니다.
